1°git pull da branch principal
2° gerar uma nova branch a partir da branch principal
3° trabalhar e adicionar novas funcionalidades na nova branch que criou
4°  finalizar o trabalho na branch temporária
5° git checkout na branch principal
6° git pull
7° mergiar(Unir) o código da branch temporária com a branch principal (depois de testar)
8° git push da branch principal

git init // inicializa um novo repositório 
git add . //adiciona os arquivos atuais ao próximo commit 
git status //verificar o status atual dos repositório git
git commit -m "mensagem do commit" // cria um novo commit com uma mensagem
git push // envia as atualizações para a nuvem na branch atualmente ativa
git branch  //permite listar e ver qual branch está ativa atualmente 
git checkout nome-da-branch // permite mudar para uma nova branch
git checkout -b "nome da branch de origem" "nome da nova branch" //permite mudar e criar uma
								 nova branch com base em 
										outra
git merge "branch a receber merge" // permite fazer o merge da branch ativa atualmente com
					outra branch
git pull // atualiza a branch atualmente ativa
